# tastamp-light Roadmap

> 이 파일은 프로젝트의 단일 진실 공급원(Single Source of Truth)입니다.
> Claude(및 기타 AI 모델)는 세션 시작 시 이 파일을 읽고, 유저 지시 없이는 완료 체크 외 임의 편집을 하지 않습니다.

---

## 📋 현재 상태

- **최근 업데이트:** 2026-05-12
- **활성 작업:**
  1. 국내 서비스 신뢰도 강화 (기여자 표시 및 신고 시스템)
  2. 운영 효율화를 위한 디스코드 웹훅(Webhook) 연동
  3. 해외 확장을 위한 데이터 어댑터(IPlaceStandard) 설계
- **블로커:** Google Places API 과금 리스크 (리스트+딥링크 전략으로 우회 결정)

---

## ✅ 완료된 기능

- [x] 프로젝트 초기 세팅 (Next.js + Supabase + Naver Maps)
- [x] 기본 지도 UI 및 모바일 하단 내비게이션 구축
- [x] 모바일 최적화: 바텀 시트 깜빡임 현상 및 지도 백화 현상 해결
- [x] 장소 추가 기능 분리: /add 독립 페이지 라우팅 구현
- [x] 리스트 페이지 고도화: 탭 분리(전체/종류별/지역별) 및 아코디언 UI
- [x] 이미지 확대 기능: 모바일 사진 터치 시 라이트박스(모달) 제공
- [x] 닉네임 관리: 프로필 내 닉네임 변경 및 기존 데이터 동기화 로직

## 🚧 진행 중

### 🛡️ 운영 및 커뮤니티 강화 (Domestic)

**구현 방향:** 유저 참여형 데이터 검증 및 관리자 운영 효율화

**태스크:**
- [x] 디스코드 신고 웹훅: 장소/댓글 신고 시 관리자 디스코드 채널 알림
- [x] Vercel Web Analytics 연동 (트래픽 추적)
- [x] 기여자(Contributor) 표시: 장소 카드/상세에 `Added by [닉네임]` 명시
- [x] 소셜 리액션: 장소별 직접적인 '좋아요/싫어요' 기능
- [ ] 보틀 정보 매칭: 특정 장소에서 구매 가능한 보틀 리스트 연동

---

### 🌏 해외 지역 지원 (Global Expansion)

**구현 방향:** 지도 렌더링 비용을 절감하는 '리스트 기반 MVP' 전략

**세부 논의:**
- 해외 탭은 지도를 그리지 않고(0원), 구글 맵 앱으로 아웃링크(Deep Link)
- 월별 해외 장소 저장 횟수 제한(Quota) 기반 구독제($2~3) 도입

**태스크:**
- [ ] `src/lib/adapters/`: IPlaceStandard 인터페이스 및 API 파서 구축
- [ ] DB 스키마 업데이트: `region`, `source`, `usage_quota` 컬럼 추가
- [ ] 구글 Places API 세션 토큰 및 필드 마스크 최적화 적용

---

## ✅ Phase 2: 커뮤니티 및 프로필 고도화

- [x] 태그 기여자(Tag Attribution) 추적: `tags.added_by` 컬럼 추가, 신규 태그 등록 시 user_id 기록
- [x] 마이페이지 탭 구조 도입: 장소 / 코멘트 / 사진 / 즐겨찾기 탭 전환 UI
- [x] 마이페이지 데이터 페칭: submitted_by·user_id 기준 개인화 데이터 일괄 조회 (Supabase join 활용)

---

**세부 논의:**

**태스크:**
